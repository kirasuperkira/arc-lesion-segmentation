## How to report a bug

1. Make sure the bug hasn't already been reported in the [Issues](../../issues) section.
2. Create a new Issue tagged `bug`.
3. Include in the description:
   - operating system
   - GNU Octave version and installed package versions
   - a detailed description of the problem
   - steps to reproduce
   - the full error message text

## How to suggest an improvement

Create an Issue tagged `enhancement`. Describe:
- exactly what you propose to change or add
- the rationale and expected effect

## Workflow (Pull Request Workflow)

```
1. Fork the repository
2. Create a branch for your change:
       git checkout -b feature/your-feature-name
3. Make changes in your local copy
4. Test the changes on sample data
5. Open a Pull Request describing:
   — what was changed
   — why it's needed
   — how it was tested
```

## Ethical requirements

By contributing to this project, you agree to follow the ethical standards for working with medical data:
- do not attempt to re-identify participants
- use the data exclusively for scientific and educational purposes
