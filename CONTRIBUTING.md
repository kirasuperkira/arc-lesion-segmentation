## Как сообщить об ошибке

1. Убедитесь, что ошибка ещё не зафиксирована в разделе [Issues](../../issues).
2. Создайте новый Issue с тегом `bug`.
3. Укажите в описании:
   - операционную систему
   - версию GNU Octave и установленных пакетов
   - подробное описание проблемы
   - шаги для воспроизведения
   - полный текст сообщения об ошибке

---

## Как предложить улучшение

Создайте Issue с тегом `enhancement`. Опишите:
- что именно вы предлагаете изменить или добавить
- обоснование и ожидаемый эффект

---

## Рабочий процесс (Pull Request Workflow)

```
1. Сделайте Fork репозитория
2. Создайте ветку для вашего изменения:
       git checkout -b feature/your-feature-name
3. Внесите изменения в локальную копию
4. Протестируйте изменения на тестовых данных
5. Создайте Pull Request с описанием:
   — что изменено
   — зачем это нужно
   — как протестировано
```

## Этические требования

Участвуя в проекте, вы соглашаетесь соблюдать этические нормы работы с медицинскими данными:
- не предпринимать попыток повторной идентификации участников
- использовать данные исключительно в научных и образовательных целях
